# 简单工厂模式(Simple Factory)

## 原理
通常一组拥有共同父类的类，创建的过程不同，但是创建的结果是相同（面向接口编程，New返回接口），这时可以通过简单工厂模式负责实例化其他类，实现把类的创建延迟到子类，并且不关心创建的细节


## 适用场景
- 工厂类负责创建的对象比较少，减少不必要的判断逻辑。
- 客户端只需知道传入工厂类的参数，而不不关心对象的创建细节。


## 例子
类库实例化不同的加密算法
- IEncrypt(创建接口)
- MD5（MD5算法类）
- SHA1（SHA1算法类）
- CRC32（CRC32算法类）
- EncryptFactory（算法工厂）
